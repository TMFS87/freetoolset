❓ FAQ

What is a hash?

A hash is a fixed-length string generated from input data using a mathematical algorithm. The same input always produces the same hash, but even a tiny change to the input produces a completely different output. Hashes are one-way functions — you cannot reverse them to recover the original data.

Can a hash be reversed or decrypted?

No. Hash functions are designed to be one-way. It is computationally infeasible to derive the original input from a hash. The only way to "crack" a hash is via brute-force or rainbow-table attacks, which try many inputs until a matching hash is found. This is why strong passwords with good hashing algorithms are very secure.

What is the difference between SHA-256 and MD5?

MD5 produces a 128-bit hash (32 hex characters) and is considered cryptographically broken — it is vulnerable to collision attacks. SHA-256 produces a 256-bit hash (64 hex characters) and is currently considered secure for most applications. For password storage, file integrity verification, or digital signatures, always prefer SHA-256 or stronger. MD5 should only be used for non-security-critical checksums.

Which algorithm should I use?

For security-sensitive applications (password hashing, digital signatures, certificates), use SHA-256 or SHA-512. For simple file checksums or legacy compatibility, MD5 or SHA-1 may suffice. Note that SHA-1 is also deprecated for security use due to known vulnerabilities. When in doubt, choose SHA-256.
