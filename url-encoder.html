🔗 URL Encoder / Decoder

Encode URLs to percent-encoded format or decode them back to readable text. Supports both encodeURI and encodeURIComponent modes.

🔒 All processing happens in your browser. No data is uploaded to any server.

📋 How to Use

  1. Select the mode: "Encode" (text to URL-encoded) or "Decode" (URL-encoded to text).
  2. When encoding, choose the method: encodeURIComponent for query parameters or encodeURI for full URLs.
  3. Type or paste your text into the input box.
  4. Click "Encode" or "Decode" to process, or use "Swap" to flip input and output.
  5. Click "Copy Result" to copy the output to your clipboard.

❓ FAQ

What is URL encoding?

URL encoding (also called percent encoding) converts characters that are not allowed in a URL — such as spaces, special symbols, and non-ASCII characters — into a format that can be safely transmitted. Each unsafe character is replaced with a percent sign followed by two hexadecimal digits, e.g. a space becomes %20.

Why do I need to encode URLs?

URLs can only contain a limited set of ASCII characters. Characters like spaces, &, =, ?, and non-English letters must be encoded to avoid ambiguity and ensure the URL is interpreted correctly by servers, browsers, and APIs. Without encoding, special characters could break the URL structure or alter its meaning.

What is the difference between encodeURI and encodeURIComponent?

encodeURI is designed for encoding a complete URL. It preserves characters that have special meaning in a URL — such as : / ? # & = + — so the URL structure stays intact. encodeURIComponent is meant for encoding individual query parameter values. It encodes all reserved characters, ensuring the value can be safely embedded in a query string. Use encodeURI for full URLs and encodeURIComponent for parameter values.

Does it support Unicode and emoji?

Yes. Both encodeURI and encodeURIComponent in modern browsers handle UTF-8 encoding, so Chinese, Japanese, Korean characters, emoji, and any other Unicode text are correctly percent-encoded (e.g. "中" becomes %E4%B8%AD).
