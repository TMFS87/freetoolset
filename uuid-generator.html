❓ FAQ

What is a UUID?

A UUID (Universally Unique Identifier) is a 128-bit identifier represented as a 36-character string in the format xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx, where each "x" is a hexadecimal digit. UUIDs are designed to be globally unique without requiring a central coordinating authority, making them ideal for distributed systems, databases, and APIs.

What is UUID v4?

UUID version 4 (v4) uses random or pseudo-random numbers to generate the identifier. Except for the version and variant bits (which identify it as a v4 UUID), all 122 remaining bits are random. This tool uses the browser's crypto.getRandomValues() function, which provides cryptographically secure random values. UUID v4 is the most commonly used version in modern applications.

Can UUIDs collide or repeat?

The probability of a UUID v4 collision is astronomically small. With 122 random bits, there are 2¹²² (approximately 5.3 × 10³⁶) possible values. To have a 50% chance of a single collision, you would need to generate about 2.71 × 10¹⁸ UUIDs. In practice, UUID collisions are virtually impossible and can be safely ignored for any real-world application.

What are UUIDs used for?

UUIDs are widely used as primary keys in databases, identifiers for distributed systems, session tokens, file names, API request IDs, and anywhere a globally unique identifier is needed without coordination between systems. They are especially useful in microservices architectures and event-driven systems.
