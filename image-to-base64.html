❓ FAQ

What is Base64?

Base64 is an encoding scheme that converts binary data into a text string using 64 ASCII characters (A–Z, a–z, 0–9, +, /). It allows binary data — like images — to be represented as text, making it possible to embed images directly in HTML, CSS, JSON, XML, and other text-based formats.

What is converting images to Base64 useful for?

Converting images to Base64 is useful for embedding images directly in HTML or CSS (as Data URIs), reducing HTTP requests on small images, including images in JSON API responses, sending images in email templates, and storing image data in text-based configuration files or databases.

Does Base64 make the file bigger?

Yes. Base64 encoding increases file size by approximately 33% (every 3 bytes of binary data becomes 4 Base64 characters). For example, a 100KB image becomes roughly 133KB in Base64. For small icons or sprites this overhead is negligible, but for large images it's generally better to serve them as separate files to avoid inflating page size and blocking the main thread.

Is there a file size limit?

The limit depends on your browser's available memory. Most browsers can handle images up to several megabytes without issues. However, very large Base64 strings can slow down text editors and development tools, so we recommend using Base64 primarily for small images and icons.
